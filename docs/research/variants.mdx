The [returns levers](/research/returns) are implemented as **backtest-only** code so they
can be measured without touching the live `ClaudeBreakout` paper bot.

<Info>
  `config.dry.json` still runs plain `ClaudeBreakout`. None of these variants are wired into
  the live bot — they exist to A/B each lever in backtesting.
</Info>

## Strategy variants

`ftbot/strategies/breakout_research.py` — each isolates one lever for clean attribution
(composed via mixins on top of `ClaudeBreakout`):

| Strategy | Lever(s) |
| --- | --- |
| `ClaudeBreakoutRisk` | volatility-targeted unit sizing |
| `ClaudeBreakoutCorr` | + correlation / crowding down-scaling |
| `ClaudeBreakoutPyramid` | + Turtle pyramiding |
| `ClaudeBreakoutChandelier` | + chandelier trailing exit |
| `ClaudeBreakoutPro` | all levers combined |

`ftbot/strategies/breakout_hyperopt.py` — `ClaudeBreakoutHO`, which exposes the entry/exit
windows, trend EMA, ATR stop, and risk fraction as hyperopt parameters for
[walk-forward tuning](/research/backtesting).

## The math (pure, unit-tested)

`ftbot/strategies/research_sizing.py` holds dependency-free helpers — no freqtrade or
TA-Lib — so the logic is unit-tested in CI on a bare Python runner:

<AccordionGroup>
  <Accordion title="risk_based_stake" icon="scale-balanced">
    `stake = (equity × risk_fraction) / (stop_atr_mult × atr_pct)`, clamped to a notional cap
    and min/max. Equalizes dollar risk per trade across pairs.
  </Accordion>
  <Accordion title="pyramid_should_add" icon="layer-group">
    True when price has advanced a further `step_atr`×ATR in favour, up to `max_units` total.
    Losers never reach a positive threshold, so only winners scale.
  </Accordion>
  <Accordion title="crowding_factor" icon="users">
    `1 / (1 + k·n)` for `n` same-direction open trades — shrinks each new entry as the book
    crowds onto one side.
  </Accordion>
  <Accordion title="chandelier_stop" icon="anchor">
    Trailing stop from the best excursion since entry minus `n_atr`×ATR (long) / plus (short).
  </Accordion>
</AccordionGroup>

## CI

The `ftbot CI` GitHub Actions workflow runs on `claude/fqtrader-**` branches and any PR
touching `ftbot/`: it ruff-lints the helpers, byte-compiles every strategy file (syntax
check), and runs the helper unit tests — no TA-Lib or freqtrade needed.
